# Contractility

Локальный прототип для распознавания сканированных договоров и последующей
подготовки дополнительных соглашений. На первом этапе реализован проверяемый
OCR-конвейер: PDF.js отрисовывает страницу в браузере, Tesseract.js распознаёт
русский и английский текст, а результат экспортируется вместе с координатами и
контрольной суммой исходного PDF.

## Запуск в закрытом контуре

На целевом компьютере `npm install` не выполняется. Репозиторий уже содержит
необходимые браузерные файлы, WebAssembly-ядро и языковые модели в
`public/vendor`.

Требования времени выполнения:

- Node.js 22 или новее;
- актуальный Chrome, Edge или Firefox с WebAssembly и Web Workers.

```bash
node scripts/doctor.mjs
node src/server.mjs
```

После запуска откройте `http://127.0.0.1:4317`.

Сервер использует только стандартную библиотеку Node.js, слушает loopback-адрес
и не предоставляет API загрузки документов. PDF выбирается непосредственно в
браузере и не отправляется во внешнюю сеть.

## Возможности текущего MVP

- открытие локального PDF без загрузки на сервер;
- использование готового текстового слоя, если он присутствует;
- OCR сканированных страниц на русском и английском языках;
- настройка 160, 220 или 300 DPI;
- автоматическая попытка исправить поворот страницы;
- текст, уверенность OCR и координаты строк по каждой странице;
- ручное исправление распознанного текста;
- экспорт `*.ocr.json` и `*.ocr.txt`;
- SHA-256 исходного PDF и vendored-компонентов;
- полностью локальные модели без CDN.

```bash

```

## Диагностика на целевом компьютере

Если страницы завершаются с ошибкой внутри браузерного OCR, сначала выполните
обычные проверки, затем запустите диагностическую страницу:

```bash
npm test
node scripts/doctor.mjs
npm run diagnostics
```

В Safari откройте `http://127.0.0.1:4317/diagnostics.html`, выберите проблемный
PDF, дождитесь завершения и скачайте `*.diagnostics.json`. Проверка обрабатывает
только первую страницу на 220 DPI, повторяет точный вызов приложения, проверяет
структуру OCR-блоков и постобработку строк. При ошибке распознавания она также
проверяет передачу изображения через Data URL. Отчёт содержит версию Safari,
этапы и стеки ошибок, но не содержит изображение или распознанный текст.
Если Safari падает уже на `PDF.js getTextContent`, проверка завершается сразу:
основное приложение должно пропустить текстовый слой и перейти к обычному OCR.

## Обновление vendored-компонентов

Команды этого раздела выполняются только на контролируемой машине сборки с
доступом к npm, а не на целевом компьютере:

```bash
npm install --ignore-scripts
npm run vendor
npm test
```

`npm run vendor` копирует только требуемые runtime-файлы и создаёт
`public/vendor/manifest.json` с SHA-256 каждого файла. Версии зафиксированы в
`package.json` и `package-lock.json`. Сведения о лицензиях приведены в
`THIRD_PARTY_NOTICES.md`.

## Формат результата

Экспорт использует схему `contractility.ocr.v1`. Для каждой страницы хранятся:

- источник текста (`pdf-text` или `tesseract`);
- текст и уверенность;
- время обработки;
- поворот страницы;
- размеры изображения, переданного OCR;
- нормализованные координаты распознанных строк;
- признак ручного исправления.
